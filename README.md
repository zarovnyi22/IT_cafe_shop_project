# Система обліку кав'ярні

Повноцінний приклад POS/складського обліку для невеликої кав'ярні з двома ролями: **Barista** та **Admin**. Стек: Node.js + Express + Sequelize (SQLite) і фронтенд на React (Vite).

## Запуск
1. Встановіть залежності бекенду: `npm install`
2. Встановіть залежності фронтенду: `npm install --prefix frontend`
3. Запустіть сервер (створення БД та наповнення відбудеться автоматично): `npm start`
4. У другому терміналі запустіть фронтенд: `npm run dev --prefix frontend`
5. Відкрийте інтерфейс: http://localhost:5173

### Облікові записи
- Адмін: `+380991234567` / `admin123`
- Бариста: `+380997654321` / `barista123`

## Можливості
- Авторизація за телефоном та паролем з JWT і розмежуванням ролей.
- Бариста: швидке додавання товарів у кошик, оформлення продажу з автоматичним списанням інгредієнтів та вибором методу оплати.
- Адмін: реєстрація нових барист/адміністраторів, додавання інгредієнтів/продуктів (API), перегляд продажів за день/тиждень/місяць.
- Складський облік: перегляд залишків із підсвічуванням критичних запасів.

## Структура
- `server.js` — HTTP API, автентифікація, облік продажів/складу.
- `server/` — підключення до Sequelize, моделі та seed-дані, що відповідають SQL із завдання.
- `frontend/` — застосунок React (Vite) з інтерфейсом POS і адміністративним блоком.
- `coffee_shop_db.sql` — початкова схема з методичних вказівок (для звірки).
